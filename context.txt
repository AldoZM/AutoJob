== WHAT IS PROJECT ==
AutoJob = CLI bot para postularse automáticamente a empleos en OCC (y futuras plataformas).
Stack: Python + Playwright + Typer + Rich
Repo: https://github.com/AldoZM/AutoJob

== OBJETIVO ==
Automatizar postulaciones en bolsas de trabajo.
Usuario corre: python main.py apply --search "Puesto" --location "Ciudad" --limit N
Bot busca vacantes, entra a cada una, da click en "Postularme", registra en CSV.

== TECH STACK ==
- playwright: controla Chromium (no instala browser del sistema, descarga el suyo)
- typer: CLI commands (login, apply)
- rich: output con colores y tablas
- python-dotenv: lee .env para credenciales
- Chromium: descargado en ~/.local/ms-playwright/ via `playwright install chromium`

== ESTRUCTURA ==
base_bot.py       — clase abstracta, CSV logging
browser_manager.py — Playwright browser con user-agent y viewport
main.py           — CLI: login + apply commands, auto-login desde .env
occ_bot.py        — implementación OCC
requirements.txt  — playwright, typer, rich, python-dotenv
.env.example      — plantilla de credenciales
.env              — credenciales reales (gitignoreado)
auth_state.json   — sesión guardada de Playwright (gitignoreado)
postulaciones.csv — log de postulaciones (gitignoreado)

== OCC — DETALLES TÉCNICOS ==
Búsqueda URL: https://www.occ.com.mx/empleos/trabajo-en-{query}/en-{location}/
Job cards: elementos con atributo data-id en la listing page (no son <a>, son <div>)
Job detail URL: https://www.occ.com.mx/empleo/oferta/{job_id}
Apply button: button:has-text("Postularme")
Anti-bot: OCC detecta headless sin user-agent → no renderiza cards
Fix: BrowserManager siempre usa user-agent Chrome real + viewport 1280x800
